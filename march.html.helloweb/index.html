    (Topic of the month : DB2MON )

    1. BASIC SETTINGS -Setting required for getting reports.
  • db2 update db cfg using MON_ACT_METRICS EXTENDED (OR ATLEAST BASE) ;
  • db2 update db cfg using MON_REQ_METRICS EXTENDED (OR ATLEAST BASE) ;
  • db2 update db cfg using MON_OBJ_METRICS EXTENDED (OR ATLEAST BASE) ;
     
    2. DB2MON SETTINGS - db2mon can be used for both UNIX based system ,as well as Windows ;
  • For db2 v11 onwards -db2mon is included in db2 installation media. It is available /sqllib/samples/perf.
  • For below versions - you can use following link to downloads ;
  • https://www.ibm.com/support/pages/node/556789
  • For using in windows DB2 - From the above link, download db2mon script tar file for your version and extract on the system where you will collect.
  • On the same directory where 'db2mon,sql' is located, create the windows script 'db2mon.cmd' with the following code.
  • set DBNAME=%1
  • db2 -v connect to %DBNAME%
  • db2 -v create bufferpool db2monbp
  • db2 -v create user temporary tablespace db2montmptbsp bufferpool db2monbp
  • db2 +c -tvf db2mon.sql
  • db2 -v commit work
  • db2 -v connect reset
  • db2 -v connect to %DBNAME%
  • db2 -v drop tablespace db2montmptbsp
  • db2 -v drop bufferpool db2monbp
  • db2 -v connect reset
  • Open new 'db2cmd' terminal window and go to the directory where 'db2mon..cmd' exists.
  • Run 'db2mon.cmd' db2mon.cmd >> report.txt

    2. Studying db2mon report: -
  • This report should be collected periodically ,when the issue is persisting
  • Various metrics
  • Use the "Throughput metrics at database level" section to identify work that is done by the system.
  • Use the "Time breakdown at database level" section to identify processing time for the database.
  • The "Wait times at database level" section reports the time that the database spends waiting. The "Wait times at database level" section is often the most important top-level information to assess performance health.
  • The "Top SQL statements by execution time" section shows raw CPU usage. There are various other sections - which can help you find bottleneck in your db and enviornment.
  • The below link has complete details
  • https://www.ibm.com/docs/en/db2/11.5?topic=tuning-collecting-reporting-performance-monitor-data.html
