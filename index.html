    (Topic of the month : ADMIN_MOVE_TABLE )

    1. BASIC INFO -Its definition.
  • It is a stored procedure that can be used to perform an online move of a table, while transactions are still occurring against the table. It uses triggers against the source table to track changes in a staging table while moving the data to a new table. Once the new table is populated, then the data tracked by the triggers is replayed against the target table and a table-level exclusive lock is obtained on the source table for a short period to make the switch (rename).

    •  
    2. BASIC STEPS OF ADMIN_MOVE_TABLES - The ADMIN_MOVE_TABLE operation can be divided in 5 stages ;
  • INIT--->COPY--->REPLAY--->SWAP--->CLEANUP
  • To understand more details on it ,Please check below link:
  • https://www.ibm.com/docs/en/db2woc?topic=procedure-admin-move-table-move-tables-online
  • Here is one of example , Where we are try to move the table from One tablespace to another:
  • > db2 "select substr(tabname, 1,20) as tabname, substr(tabschema,1,20) as tabschema, substr(TBSPACE,1,20) as tablespace from syscat.tables where tabname='EMPLOYEE'"
  • TABNAME TABSCHEMA TABLESPACE
  • -------------------- -------------------- --------------------
  • EMPLOYEE PROFESSIONAL USERSPACE1
  • 1 record(s) selected.
  • Simple Syntax: > db2 "call sysproc.ADMIN_MOVE_TABLE ('PROFESSIONAL','EMPLOYEE','TBS1','TBS1','TBS1','','','','','','move')"
